# Order Matters: Sequence to Sequence for Sets

**Authors:** Oriol Vinyals, Samy Bengio, Manjunath Kudlur
**Year:** 2016
**Venue:** ICLR 2016
**Institution:** Google Brain

---

## Key Contributions

- Identified and addressed a fundamental problem: standard Seq2Seq models impose an **arbitrary ordering** on both input and output sets, which can significantly harm performance.
- Proposed **Read-Process-and-Write (RPW)**, an architecture based on attention mechanisms that is invariant to input ordering and can learn effective output orderings.
- Introduced the use of **set2set** (an attention-based pooling mechanism) for encoding unordered sets into fixed-length representations.
- Demonstrated that **output order matters** and that finding a good ordering for the training targets can substantially improve performance.
- Showed strong results on sorting, combinatorial optimization, and language modeling tasks.

---

## Background & Motivation

Seq2Seq models assume both the input and output are **sequences** (ordered). However, many real-world problems involve **sets** (unordered):

- **Input sets:** A collection of points, a bag of features, a set of items with no natural ordering.
- **Output sets:** Predicting a set of labels, a group of objects, or any collection where order is irrelevant.

Encoding a set as a sequence requires choosing an arbitrary order, and the model's performance can be highly sensitive to this choice. Similarly, when the output is a set, training with an arbitrary target ordering can confuse the model. This paper systematically studies these issues and proposes solutions.

## Method / Architecture

### The Problem of Order

Given a set $\{a, b, c\}$, there are $3! = 6$ possible orderings for a Seq2Seq model. If the "correct" training order is $(a, b, c)$, the model is penalized for producing $(b, a, c)$ even though they represent the same set. More generally, for a set of size $n$, there are $n!$ orderings.

### Read-Process-and-Write Architecture

The RPW model has three stages:

**1. Read (Encoding the Input Set)**

An input set $\{x_1, x_2, \ldots, x_n\}$ is first independently embedded:

$$e_i = \text{embed}(x_i)$$

**2. Process (Order-Invariant Set Encoding)**

Instead of reading embeddings sequentially, the **Process** block uses an attention mechanism that iterates $T$ times over the input embeddings to build an order-invariant representation.

At each processing step $t = 1, \ldots, T$:

$$q_t = \text{LSTM}(q_{t-1}^*)$$

$$e_{it} = f(e_i, q_t) \quad \text{(attention energy)}$$

$$\alpha_{it} = \frac{\exp(e_{it})}{\sum_j \exp(e_{jt})}$$

$$r_t = \sum_i \alpha_{it} e_i$$

$$q_t^* = [q_t; r_t]$$

where $q_t$ is the query (LSTM state) and $r_t$ is the attended readout. After $T$ steps, $q_T^*$ summarizes the entire set in an order-invariant way.

This is the **set2set** mechanism: the LSTM iteratively attends to the input set, accumulating information regardless of input ordering.

**3. Write (Decoding the Output)**

The output is generated by a decoder (another LSTM with attention) conditioned on the process block's output. The decoder uses a Pointer Network-style mechanism if the output should reference input elements.

### Handling Output Order

For **output sets**, the paper proposes two approaches:

**Approach 1: Train with best ordering.**

Find the target ordering that maximizes the model's likelihood:

$$\pi^* = \arg\max_{\pi \in \Pi} p(y_{\pi(1)}, y_{\pi(2)}, \ldots, y_{\pi(n)} \mid x; \theta)$$

In practice, this combinatorial search is approximated (e.g., greedy matching or beam search over orderings).

**Approach 2: Let the model learn the ordering.**

Train with a default ordering but allow the model to discover its own preferred ordering through the Process mechanism and flexible decoding.

### Set2Set Pooling

For tasks requiring a fixed-size representation of a set, the set2set mechanism provides a more expressive alternative to simple sum/mean/max pooling:

| Pooling Method | Expressiveness | Order Invariant |
|:--------------|:--------------:|:---------------:|
| Sum pooling | Low | Yes |
| Mean pooling | Low | Yes |
| Max pooling | Medium | Yes |
| **Set2Set (attention)** | **High** | **Yes** |

## Key Results

### Sorting Task

Given a set of $n$ numbers, output them in sorted order. This tests the model's ability to handle input sets and produce an ordered output.

| Model | Accuracy (n=5) | Accuracy (n=15) |
|:------|:--------------:|:---------------:|
| Seq2Seq (arbitrary input order) | 72.4% | 0.2% |
| Seq2Seq (sorted input) | 98.7% | 11.5% |
| **RPW (Process + attention)** | **98.9%** | **17.2%** |

The RPW model achieves strong performance regardless of input order.

### Effect of Output Ordering

On a task where the output is a set, training with different orderings produces very different results:

| Output Order Strategy | Log-Likelihood |
|:---------------------|:--------------:|
| Random ordering | -7.8 |
| Fixed arbitrary ordering | -6.2 |
| Sorted ordering | -5.1 |
| Best ordering (searched) | **-4.3** |

Finding a good output ordering during training can improve results by a large margin.

### Language Modeling (Word Ordering)

Given a bag of words, reconstruct the original sentence. The task evaluates the model's ability to impose structure on an unordered input.

The RPW model with the Process block significantly outperformed a standard Seq2Seq model, demonstrating the value of order-invariant input processing.

## Impact & Legacy

- **Set2Set became standard for graph/set tasks:** The attention-based set encoding was widely adopted in graph neural networks (e.g., for molecular property prediction) and point cloud processing.
- **Influenced Deep Sets and Set Transformers:** Directly inspired subsequent work on permutation-invariant architectures, including Deep Sets (Zaheer et al., 2017) and Set Transformer (Lee et al., 2019).
- **Output ordering remains relevant:** The insight that output ordering matters for training set-output models continues to be relevant in object detection (DETR), molecule generation, and other tasks.
- **Multi-step attention processing:** The iterative attention mechanism in the Process block foreshadowed multi-head and multi-layer attention in Transformers.
- **Combinatorial optimization:** Extended the neural combinatorial optimization line started by Pointer Networks, showing that order-invariant processing is key for set-structured problems.

---

## Key Takeaways

1. Input order matters for Seq2Seq models: simply permuting the input set can dramatically change performance, making order-invariant processing essential.
2. Output order matters too: when the target is a set, training with a carefully chosen ordering (or the best ordering) significantly outperforms random or arbitrary ordering.
3. The set2set mechanism (iterative attention over the input) provides a powerful, order-invariant way to encode sets into fixed-length representations.
4. The Read-Process-and-Write architecture decouples set encoding (order-invariant) from sequence decoding (order-dependent), providing a clean framework for set-to-sequence problems.
5. Simple architectural choices about ordering can have surprisingly large effects on model performance -- a lesson that extends far beyond the specific tasks in this paper.
